# 🍒 チェリー方式チェッカー まとめ

## 🎯 1. このシステムの「役割」

芸漫のエントリーにおいて、**「チェリー方式（同じ組み合わせでの出演は1枠のみ、別ユニットなら最大2枠まで出演できる特殊ルール）」**の複雑な集計・検知を**完全自動化**するシステム。
人の目では絶対に見落としがちな重複やルール違反を、入力した瞬間にすべて自動で叩き潰して、ライブ運営の集計ミスをなくす。

---

## 🛠️ 2. 搭載されている「全機能一覧」

### ① 出演形式のワンタップ選択（Step 1）
* **ピン / コンビ / トリオ** をボタン一つで切り替え、入力欄を自動生成。
* **「それ以上」** を選べば、4人以上の大人数ユニット（カルテットなど）の人数も自由に指定可能。

### ② 学籍番号ベースのリアルタイム判定（Step 2）
同姓同名による誤判定を防ぐため、**「学籍番号」をキーにして**チェリー方式の全ロジックを裏側で自動計算。
全角入力による判定ミスを防ぐための注意書き付き。

### ③ 鉄壁の「3大エラー防壁」機能
演者がルールを誤解して申し込んできた場合、画面に理由を赤文字でハッキリ表示して登録を未然に防ぐ。
1. **【3枠目出演エラー】**：すでに2枠出演が確定している本人が、3枠目を申し込んだら即座にブロック。
2. **【連鎖ブロックエラー】**：相方が2枠出演してロックされたため、道連れで出演できなくなった「過去の相方」のエントリーをブロック。（誰の影響で出られないかの理由も表示）
3. **【未来予知（チェリー方式違反）】**：まだ誰も2枠出ていない状態であっても、「そのエントリーを許可すると、同じユニット内に2回出演する人が複数並んでしまう（中抜き・横スライドの禁忌）」という未来のルール違反を予知してブロック。

### ④ ガチガチの「プライバシー保護」機能
登録済みリストや出演不可者リストには、**「名前」だけを表示し、個人情報である「学籍番号」は完全に非表示**に。

### ⑤ 「隠し学籍番号検索」機能
画面上は学籍番号が隠れているが、検索窓に学籍番号を打ち込むと、裏側のデータと照合して「その学籍番号の人が出ているユニット」が炙り出される。（名前やユニット名での検索も可能）

### ⑥ バリデーション（入力ミス防止）機能
ユニット名の入力忘れを防止。
学籍番号や名前の入力漏れを防止。
「1つのユニット内に、同じ学籍番号を2回入力する」という身内での重複ボケをアラートで阻止。

### ⑦ 運営用「Excel・CSV出力」機能
ボタン一つで、文字化けのない綺麗なCSVファイルをダウンロード。
画面上では隠されていた「学籍番号」がCSVにはしっかり記載されるため、主催者の事務処理やExcelでの並び替え・プログラム組みにそのまま使える。

### ⑧ 状態保持（Local Storage）機能
ブラウザの記憶領域に自動保存されるため、**画面を閉じたりPCを再起動したりしても、登録データが消えずに**そのまま残る。
「すべて削除」ボタンで一発で初期化（0組）することも可能。
